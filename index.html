Badass Server Mocks

Get Started Watch Screencast

Make client side development fun

When building web applications, you are in a dynamic environment. In most cases you are not alone working on the project, you are dependent on implementations of server features which you need to wait for to finish your feature.

BDSM allows you to continue coding as if the server was ready using easily created mocks while testing various scenarios of server responses or delays so you can continue to finish your feature.

You don't need to change anything in your code once the server is ready, just turn off your mock and everything is good to go.

Getting Started


  # Install BDSM in your project
  $ npm install --save-dev bdsmjs

  # Require BDSM in your application
  require('bdsmjs');

When done you should see the BDSM icon on the lower right corner of your application.

Easy Testing

No need to replace network dependencies, just set a mock programatically and continue writing your test normally.

Keeping in sync

Easily share your mocks across your team using your version control system.
